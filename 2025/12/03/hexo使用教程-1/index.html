hexo使用教程

指令 完整格式 核心作用 适用场景 补充说明
npx hexo init npx hexo init [文件夹名] 初始化 Hexo 博客项目 首次创建博客时 1. 不指定文件夹名则在当前目录初始化；2. 要求目标目录为空（非空需先备份删除文件）；3. 自动生成 _config.yml、source、themes 等核心文件
npx hexo init --force - 强制在非空目录初始化 需重建核心文件（如 package.json 丢失） 仅覆盖核心依赖文件，不删除 _config.yml、source 等配置 / 文章内容（部分版本可能不支持，需谨慎使用）
npm install - 安装项目依赖 1. 初始化项目后；2. 复制他人项目后；3. 依赖丢失 / 冲突后 读取 package.json 中的依赖清单，自动下载到 node_modules 文件夹
npm install [插件名] --save - 安装 Hexo 插件 需要扩展功能时（如部署、评论） 示例：npm install hexo-deployer-git --save（安装 Git 部署插件），安装后自动更新 package.json 依赖清单
npx hexo g npx hexo generate 生成静态博客文件 发布前构建博客内容 1. 把 Markdown 文章、主题模板编译为 HTML/CSS/JS；2. 生成的文件存于 public 文件夹；3. 后续部署的核心素材来源
npx hexo d npx hexo deploy 部署静态文件到服务器 生成文件后发布到线上 1. 需先在 _config.yml 配置 deploy 信息（如 GitHub 仓库）；2. 依赖对应部署插件（如 hexo-deployer-git）；3. 自动推送 public 文件夹内容到目标平台（如 GitHub Pages）
npx hexo g && npx hexo d - 生成 + 部署一键执行 快速发布博客更新 && 表示前一个命令执行成功后自动执行下一个，避免手动分步操作
npx hexo s npx hexo server 启动本地预览服务器 编写文章 / 修改主题后预览效果 1. 默认访问地址：http://localhost:4000；2. 实时刷新内容（修改文章 / 配置后无需重启，刷新浏览器即可）
npx hexo new [文章标题] npx hexo new "我的第一篇博客" 创建新博客文章 撰写新内容时 1. 文章默认生成在 source/_posts 文件夹；2. 格式为 Markdown（.md 文件）；3. 自动生成文章头部信息（标题、日期、分类等）
npx hexo new page [页面名] npx hexo new page "about" 创建独立页面 需要添加 “关于我”“分类” 等页面时 1. 页面生成在 source/[页面名] 文件夹；2. 适用于创建非文章类静态页面（如导航栏菜单页面）
npx hexo clean - 清理生成的静态文件 部署失败 / 内容更新不生效时 1. 删除 public 文件夹和缓存文件；2. 解决 “修改内容后预览 / 部署无变化” 的缓存问题，清理后需重新执行 hexo g 生成
hexo -v - 查看 Hexo 版本 排查版本兼容问题时 需全局安装 Hexo（npm install -g hexo-cli）后才能直接使用，否则需用 npx hexo -v
